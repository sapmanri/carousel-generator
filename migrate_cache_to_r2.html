캐시 마이그레이션 — GitHub image_cache.json → R2 분리 구조

cache/index.json + cache/images/<hash>.json
⚠️ 이 도구는 읽기는 GitHub에서, 쓰기는 R2에만 합니다. 기존 GitHub image_cache.json은 절대 수정/삭제하지 않습니다 (안정화 전까지 읽기 전용 백업으로 유지). R2 쪽에 이미 같은 키가 있으면 덮어씁니다 — 재실행해도 안전(idempotent)하지만, 실행 전 1단계 미리보기로 먼저 확인하세요.

0. 설정 확인

GitHub 토큰, R2 자격증명이 이 브라우저에 저장되어 있어야 합니다 (다른 도구의 설정과 공유됨).
확인 중…

1. GitHub 데이터 미리보기 (읽기 전용, 안전)

image_cache.json을 읽어서 entry 개수와 샘플만 보여줍니다. 아무것도 쓰지 않습니다.

2. 마이그레이션 실행 (R2에 씀)

각 entry를 cache/images/<hash>.json으로 R2에 저장하고, 가벼운 필드만 모은 cache/index.json을 한 번에 저장합니다. 1단계를 먼저 실행해서 개수를 확인한 뒤 진행하세요.

3. 검증

GitHub entries 수 / R2 cache/images/ 객체 수 / cache/index.json entries 수, 세 개가 정확히 일치해야 통과입니다.
